>[!WARNING|style: flat|label: 简要说明]
>
><span style='color:Blue'>问：在什么情况下会导致消息被重复消费呢？</span>
>
>- [`A`] 生产端原因：生产者重复发送同一条消息到消息队列`MQ`<span style='color:red'>[ 例如业务接口被多次调用，且未做幂等性处理 ]</span>
>
>- <span style='color:Blue'>[`B`] 服务端原因</span>：消费者已处理消息但在发送确认`Ack`前 →`MQ`服务异常( 如宕机 ) <span style='color:red'>[ 导致消息未被标记为已消费，`MQ`恢复后再次投递该消息 ]</span>
>
>- <span style='color:Blue'>[`C`] 消费端原因</span>：消费者处理完消息但在发送确认`Ack`前 → 自身异常( 如崩溃 ) <span style='color:red'>[`MQ`未收到确认，消费者恢复后会重新投递该消息 ]</span>
>
>---
>
><span style='color:red'>[ 解决方案 ]</span>
>
>- <span style='color:Blue'>[ 生产者 ] 发送消息时利用`Redis.LUA`判断是否重复发送 (`Redis`未存在插入，存在：说明已重复发送 )</span>
>
>- <span style='color:red'>[ 消费者 ] 收取到消息业务处理完成后`ACK`[ 同时将`Redis`中该消息标记为已完成，防止因`MQ`宕机重复发送 ] </span>
>
>  <span style='color:red'>[ 关键：如`MQ`宕机，导致重复发送则根据`Redis`消息完成状态 - 跳过业务直接`ACK`]</span>
>
